# 发布说明与版本策略

最后更新：2026-07-17

## 目标
- 让版本号、Release、提交节奏更稳定
- 方便毕业设计阶段展示、周报记录、导师沟通与后续封装交付

## 当前版本策略

### 正式版
- 适合：
  - 一个大阶段已经稳定
  - 核心功能、文档、演示流程基本完整

示例：
- `v1.5.0`
- `v1.6.0`

### Beta 版
- 适合：
  - 架构升级较大
  - 已能演示，但仍在联调
  - 可能还有交互或边角问题待收口

示例：
- `v2.0.0-beta.1`

## 推荐的语义

### 补丁版本
- 只修 bug
- 例如：
  - `v1.6.1`

### 次版本
- 增加一批稳定功能
- 例如：
  - `v1.7.0`

### 主版本
- 架构或产品阶段有明显跃迁
- 例如：
  - 接入数据库
  - 完成封装
  - 引入登录和角色体系

## 当前推荐节奏

### 现在
- 保持：
  - `v3.0.0-beta.x`
- `v2.0.0` 保留为最低可交付稳定基线，不移动、不覆盖
- `3.x` Beta 用于产品化主线联调，当前范围包括：
  - 登录、注册、角色与平台账号治理
  - 企业注册审批、成员管理与租户数据隔离
  - 企业路网拓扑、连续运动、冲突处理与动态避让
  - 电量自治、充电站容量与企业独立客户端
  - ComfyUI 企业素材生成桥接

### 等满足以下条件后，再转正
- 三类企业角色与平台管理员权限边界稳定
- 企业注册、审批、成员管理与作用域隔离完成端到端回归
- SQLite / MySQL 核心链路和迁移流程稳定
- 企业路网、连续运动、动态避让、电量与回充回归通过
- 综合版与企业独立客户端封装稳定
- 演示脚本、测试清单和公开文档完整

届时可升级为：
- `v3.0.0`

## GitHub Release 处理建议

### 什么时候发 draft release
- 当一个阶段已经成型
- 即使还没完全转正，也可以先发 draft

### Release 文案建议包含
- Highlights
- Added
- Improved
- Fixed
- Validation
- Notes

## 提交信息建议

### 功能
- `feat: ...`

### 修复
- `fix: ...`

### 重构
- `refactor: ...`

### 文档
- `docs: ...`

### 阶段版本
- `release:vX.Y.Z`

## 当前版本线
- 最低可交付稳定版：`v2.0.0`
- 产品化预发布线：`v3.0.0-beta.x`
- 产品化稳定目标：`v3.0.0`

## 与文档联动
- 每次版本更新建议同步检查：
  - `CHANGELOG.md`
  - `RELEASE_DRAFT_*.md`
  - `RELEASE_UPLOAD_*.md`
  - `PROJECT_IMPLEMENTATION_PLAN_v3.md`
  - `PROJECT_IMPLEMENTATION_PLAN_v4.md`
