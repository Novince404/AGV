# 答辩资料阅读顺序

这组文档的目标是让你能把系统讲成一个完整工程，而不是零散功能堆叠。建议按下面顺序准备。

## 1. 先讲系统结构

阅读 `CODE_STRUCTURE.md`，先能回答三个问题：

- 前端、后端、数据库、ComfyUI 分别在哪里。
- 地图、车辆、任务、调度、用户治理各自由哪些模块负责。
- 用户一次操作从前端到后端再到数据库的路径是什么。

## 2. 再讲算法主线

阅读 `DISPATCH_AND_ALGORITHMS.md`，重点准备：

- 障碍物如何影响路径。
- 自动调度和手动调度有什么区别。
- `simple` 和 `astar` 的思路、优缺点和适用场景。
- 企业路网拓扑如何在普通网格路径之上增加工业约束。

## 3. 补上数据闭环

阅读 `DATABASE_FLOW.md`，重点讲清：

- 为什么同时支持内存、SQLite、MySQL。
- 数据写入数据库前经过了哪些层。
- 地图、任务、车辆和 AI 作业分别落在哪类表。

## 4. 讲清 AI 素材链路

阅读 `COMFYUI_FLOW.md`，重点讲清：

- 前端如何选择素材来源。
- 后端如何创建渲染任务并调用 ComfyUI。
- 图片结果如何从 ComfyUI 回到系统。

## 5. 最后准备亮点问答

阅读 `HIGHLIGHTS_AND_QA.md`，把算法、工程、演示和扩展性问题准备成短回答。
