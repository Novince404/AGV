# AGV调度系统算法逻辑图

以下逻辑图展示了Simple算法、A*算法以及动态避让机制在多AGV调度系统中的交互关系，并标注了论文章节对应关系。

```
                   ┌───────────────────────┐
                   │     任务创建/输入      │
                   └─────────┬─────────────┘
                             │
                             ▼
                   ┌───────────────────────┐
                   │   Simple算法路径规划   │
                   │(直线候选路径计算)     │
                   └─────────┬─────────────┘
                             │
                             ▼
                   ┌───────────────────────┐
                   │      A*路径规划       │
                   │  网格启发式搜索       │
                   │  考虑动态障碍节点     │
                   └─────────┬─────────────┘
                             │
                             ▼
                   ┌───────────────────────┐
                   │  动态占用/避让检查    │
                   │ _build_live_topology_  │
                   │ edge_blockers / nodes  │
                   └─────────┬─────────────┘
                             │
                             ▼
                   ┌───────────────────────┐
                   │ 路径整合与冲突解决     │
                   │ _plan_topology_path    │
                   │ 考虑占用节点/边避开    │
                   └─────────┬─────────────┘
                             │
                             ▼
                   ┌───────────────────────┐
                   │  前端显示/AGV执行     │
                   │ App.vue轮询刷新状态    │
                   └─────────┬─────────────┘
                             │
                             ▼
                   ┌───────────────────────┐
                   │  实验/结果记录分析     │
                   │ 路径长度/任务完成时间  │
                   └───────────────────────┘

论文章节对应：
- Simple + A* 算法：系统设计 / 系统实现
- 动态占用/避让机制：系统设计 / 实验分析
- 前端显示/执行：系统实现
- 实验/结果记录：实验与分析
```

说明：
- 避让机制在A*基础上实现，不需额外算法。
- 可在论文创新点或展望部分提到未来可扩展到WHCA*或MAS多AGV协作算法。
