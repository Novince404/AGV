# 动态避让演示操作说明

## 2026-04-26 更新

- 个人端动态避让演示已切换为纯网格 A*，不再依赖企业路网拓扑。
- `dynamic_avoidance_split_map_profile_12x8.json` 只包含地图尺寸和障碍物；拓扑节点、拓扑边、节点容量仍属于企业端路网能力。
- 运行时避让分两层：规划时把其它 AGV 的当前格、运动源格、运动目标格作为临时障碍；运行中遇到同格或对向边段冲突时短暂等待，并通过 `grid_dynamic_replan` 重新规划。
- 个人端仍保持一格一格移动；企业端仍可在拓扑运行视图中使用平滑运动效果。

## 目标

这组素材用于稳定展示“调度层动态避让”，重点不是让 AGV 随机绕障，而是让多台车在主通道产生可观察的前车占用冲突，再通过上侧等待位和下侧服务支路观察等待、让行、重试和恢复。

配套文件：

- `demo/json/dynamic_avoidance_split_map_profile_12x8.json`
- `demo/json/dynamic_avoidance_split_tasks.json`

## 推荐演示顺序

1. 确认当前没有正在运行的任务。若任务队列里有旧任务，先等待完成或清理已完成任务。
2. 在 JSON / 地图工具中导入 `dynamic_avoidance_split_map_profile_12x8.json`。
3. 在地图方案列表中应用“动态避让分流演示地图 v2 12x8”。
4. 确认至少有 3 台空闲 AGV。默认三台 AGV 的初始位置已经适合触发冲突：`(2,3)`、`(5,7)`、`(1,4)`。
5. 导入 `dynamic_avoidance_split_tasks.json`。
6. 观察中部主通道、上侧等待位和下侧服务支路：车辆应在主通道前车占用时出现等待或让行，并在资源释放后恢复继续运行。

## 观察重点

- 同一物理走廊里，后车不应直接穿过前车。
- 跟随车辆进入中部窄通道时，应出现等待或保持间距，而不是直接穿过前车。
- 第三条任务主要走下侧服务支路，用来保持多车运行，但不应再全部挤死在中心或东侧入口。
- 低优先级任务在没有空闲 AGV 或前方占用时可以排队，前序任务释放后继续派发。
- 任务完成后，AGV 应回到空闲状态，不应残留 `current_edge`、旧路径或任务绑定。

## 如果现场没有触发明显避让

- 先确认任务算法是 `astar`，配套任务 JSON 已经写入该字段。
- 先确认地图确实应用了 `12 x 8` 动态避让地图，而不是只导入但未应用。
- 如果你之前导入过旧版“动态避让窄通道演示地图 12x8”，请重新导入并应用新版“动态避让分流演示地图 v2 12x8”。
- 新版地图顶部和底部不再有旧版那两个孤立单格障碍；如果仍能看到它们，说明当前应用的还是旧地图方案。
- 如果当前 AGV 位置已经被前一次演示改变，可以先创建或移动几台空闲车到走廊两端附近，再重新导入任务。
- 如果系统正在自动回仓或回充，先让这些自治动作结束，避免演示任务被回仓 / 回充行为稀释。

## 能力边界

这套演示展示的是调度层动态避让：系统基于离散网格、拓扑节点、拓扑边、容量和任务优先级处理冲突。它不等同于真实机器人传感器级避障，也不包含连续空间速度规划。后续接入真实设备时，可以在这个调度层基础上继续叠加激光雷达、视觉、DWA、ORCA、MPC 等局部避障能力。

## 2026-04-26 补充：个人端临时让行

- 个人端仍然是纯网格运行，不使用企业拓扑。
- 运行时避让现在包含三层保护：规划时把其它 AGV 的当前格、运动源格、运动目标格作为临时障碍；运动时用无向边段锁避免面对面穿车；如果低优先级或较晚任务被堵住，会搜索附近安全空格临时让行。
- 任务原因里可能出现 `grid_dynamic_replan` 或 `grid_dynamic_yield`。前者表示重新规划，后者表示车辆主动让到临时空格后再继续规划。
- 人工观察时，允许车辆短暂停顿或先向侧边/后方让一步；不应出现两台车同格、面对面穿过同一边段，或多格闪现。

## 2026-04-27 补充：真实让行复核方式

- `dynamic_avoidance_split_tasks.json` 更偏“多车分流演示”，不保证每次都形成正面对向会车。如果要专门测试真实让行，请手动制造对向任务。
- 推荐复核：在个人端 12x8 动态避让地图上，准备两台空闲 AGV，分别让它们从主通道两端相向行驶，例如 `(2,3) -> (10,3)` 和 `(10,3) -> (2,3)`，算法选择 A*。
- 预期效果：低优先级或较晚任务不应只来回挪一格，而应连续走完一段短让行路径，离开主冲突走廊后再恢复原任务。
- 三车交汇时，允许低优先级车辆排队或让行，但不应出现同格重叠、对向穿车、或长期堵死在同一个交汇格。
