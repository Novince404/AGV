# 最低可交付版演示脚本

最后更新：2026-03-18

## 演示目标
- 用一套相对稳定、时间可控的流程，展示系统已经达到“可交付毕设”的最低标准。

## 演示前准备
1. 选择一种启动方式：
   - `run_packaged_dev.bat`
   - 或 `dist\AGV_Dispatch_Package\start_agv.bat`
2. 确认页面可正常打开
3. 确认地图、点位、模板、障碍预设都能正常显示

## 推荐演示流程

### 第一部分：系统整体说明（约 1 分钟）
- 说明系统支持自动调度与手动调度
- 说明当前版本已具备数据库持久化能力
- 说明当前演示默认使用 SQLite，正式部署可切 MySQL

### 第二部分：自动任务（约 2 分钟）
1. 切换到自动模式
2. 点击地图创建单段任务
3. 展示任务被自动分配给空闲 AGV
4. 展示任务完成后状态变化

建议强调：
- 系统会根据优先级与距离自动分配 AGV
- 调度结果可实时在任务队列和地图上观察

### 第三部分：手动任务（约 1 分钟）
1. 切换到手动模式
2. 选择一台空闲 AGV
3. 点击终点并派发

建议强调：
- 手动模式体现了“指定车辆执行任务”的能力

### 第四部分：多段任务（约 2 分钟）
1. 创建一个带中转点的任务
2. 展示阶段切换
3. 展示中转点与路径显示

建议强调：
- 系统不只支持单点配送，还支持多阶段任务组织

### 第五部分：异常控制（约 2 分钟）
1. 在任务运行中执行急停
2. 再恢复
3. 视情况演示故障上报 / 故障解除
4. 视情况演示送修下线 / 恢复上线

建议强调：
- 这部分体现系统对实际调度异常的处理能力

### 第六部分：地图与数据管理（约 2 分钟）
1. 编辑障碍
2. 保存为障碍预设
3. 应用预设 / 删除预设
4. 新增一个自定义点位
5. 保存一个任务模板

建议强调：
- 当前版本已经支持点位、模板、障碍预设的持久化
- 刷新后数据仍保留

### 第七部分：数据库 / 交付说明（约 1 分钟）
- 展示当前版本支持：
  - `memory`
  - `sqlite`
  - `mysql`
- 说明当前演示采用 SQLite
- 说明系统已具备 Windows 封装试运行能力

## 演示时尽量避免的操作
- 临场切换高风险实验性功能
- 临场修改 `.env`
- 临场改 MySQL 配置
- 临场跑未验证的新功能

2. 手动单段任务
3. 自动多段任务
