# 最低可交付版人工终验执行手册 v2.0

最后更新：2026-03-18

## 目标
- 用一轮可重复、可记录的人工终验，把模块 3 从“自动化基线完成”推进到“可答辩版最终收口”。
- 本手册只关注最低可交付系统，不提前引入模块 4 之后的高风险改造项。

## 配套文档
- 执行清单：`TEST_CHECKLIST_MINIMUM_DELIVERY.md`
- 记录模板：`MANUAL_VERIFICATION_RECORD_v2.0.md`
- 自动化基线：`MINIMUM_DELIVERY_BASELINE_v2.0.md`
- 演示脚本：`DEMO_SCRIPT_MINIMUM_DELIVERY.md`

## 终验前准备

### 推荐终验顺序
1. SQLite 演示模式
2. MySQL 模式
3. 封装模式

### 启动方式
- SQLite 演示模式：`run_sqlite_dev.bat`
- MySQL 模式：`run_mysql_dev.bat`
- 封装模式：
  - 开发态封装模拟：`run_packaged_dev.bat`
  - 打包产物：`dist/AGV_Dispatch_Package/start_agv.bat`

### 执行原则
- 每跑完一组场景，就在 `MANUAL_VERIFICATION_RECORD_v2.0.md` 里记录结果。
- 如果发现阻断问题，先记现象、时间、模式、截图位置，再停止继续扩散测试。
- 如果只是轻微 UI 问题，可继续跑完整轮，再统一回收。

## 第一组：启动与基础接口

### 目标
- 确认三种启动路径都能进入系统。
- 确认基础接口、首页和地图设置数据正常。

### 操作步骤
1. 启动对应模式。
2. 确认页面能正常打开。
3. 打开 `http://127.0.0.1:8000/docs`。
4. 检查以下接口：
   - `GET /status/ui-settings`
   - `GET /status/map`
   - `GET /point/list`
   - `GET /template/list`

### 通过标准
- 页面能正常进入。
- docs 可访问。
- 以上接口都返回成功，不出现 500。

## 第二组：调度主流程

### 目标
- 确认最低可交付系统最核心的调度链路可演示。

### 场景 A：自动单段任务
1. 切换到自动模式。
2. 点击地图选择起点、终点。
3. 创建任务并等待完成。

通过标准：
- 任务成功创建。
- 成功分配 AGV。
- 路径显示正常。
- 完成后状态正常收口。

### 场景 B：手动单段任务
1. 切换到手动模式。
2. 选择一台空闲 AGV。
3. 在地图选择终点并派发。

通过标准：
- 任务成功创建。
- 指定 AGV 成功执行。
- 完成后状态正常。

### 场景 C：自动多段任务
1. 创建一个至少包含 1 个中转点的多段任务。
2. 观察阶段切换。
3. 等待任务完成。

通过标准：
- 多段任务可创建。
- 阶段切换正常。
- 中转点与路径显示正常。

### 场景 D：自动多段运行中再点地图新建任务
1. 让一个自动多段任务先运行。
2. 在其运行过程中，再次点击地图开始创建新任务。

通过标准：
- 第一下点击就显示起点。
- 起点不会瞬间消失。
- 不需要靠快速双击才能创建。

## 第三组：异常处理

### 场景 A：急停 / 恢复
1. 让任务运行起来。
2. 执行急停。
3. 再执行恢复。

通过标准：
- 急停后任务中断。
- 恢复后任务可继续。

### 场景 B：故障上报 / 故障解除
1. 对某辆 AGV 执行故障上报。
2. 查看故障列表。
3. 解除故障。

通过标准：
- 故障事件生成成功。
- 故障解除成功。

### 场景 C：送修下线 / 恢复上线
1. 送修下线一台 AGV。
2. 确认其不再参与调度。
3. 恢复上线。

通过标准：
- 下线后不再参与执行。
- 恢复后可重新参与调度。

## 第四组：地图与设置

### 场景 A：地图显示设置持久化
1. 修改小地图、图例、箭头、图标、对比显示等设置。
2. 刷新页面。

通过标准：
- 设置刷新后仍保留。
- SQLite / MySQL 模式都保持一致。

### 场景 B：障碍编辑与预设
1. 编辑一组障碍。
2. 保存为预设。
3. 应用预设。
4. 删除预设。
5. 导入布局 JSON。
6. 使用“导入另存预设”。
7. 恢复默认布局。

通过标准：
- 保存、应用、删除、导入、另存、恢复默认都正常。
- 中文提示语与布局按钮无明显错乱。

### 场景 C：地图信息区
检查以下信息是否与当前状态一致：
- 当前地图尺寸
- 当前预设名
- 障碍格数量
- 当前后端模式

## 第五组：点位与模板

### 场景 A：点位库
1. 新增自定义点位。
2. 刷新页面。
3. 删除点位。
4. 再刷新页面。

通过标准：
- 点位新增、持久化、删除都正常。

### 场景 B：模板库
1. 保存当前任务为模板。
2. 刷新页面。
3. 删除模板。
4. 再刷新页面。

通过标准：
- 模板新增、持久化、删除都正常。

### 场景 C：模板 JSON
1. 导入模板。
2. 导入文件。
3. 导出自定义模板。
4. 下载 JSON。
5. 清空模板 JSON。

通过标准：
- 功能都可执行。
- 三语言下按钮排版无明显错乱。

## 第六组：任务队列管理

### 场景 A：单任务删除
1. 删除 `pending / blocked / finished` 任务各一次。

通过标准：
- 删除成功。
- 提示语正常。

### 场景 B：运行中任务删除
1. 找到一条正常 `assigned / running` 且已绑定 AGV 的任务。
2. 点击删除。
3. 在确认弹窗中确认删除。

通过标准：
- 出现更强提醒的确认弹窗。
- 删除后任务消失。
- 绑定 AGV 停止继续执行该任务。

### 场景 C：异常任务筛选与清理
1. 切换到“只看异常”。
2. 检查是否能筛出异常任务。
3. 执行“清理异常”。

通过标准：
- 筛选正常。
- 清理正常。

### 场景 D：已完成任务批量操作
1. 执行“批量导出”。
2. 执行“批量删除”。

通过标准：
- 导出成功生成 JSON。
- 删除成功清空已完成队列。

## 第七组：多语言与界面收口

### 检查项
- 中文界面
- 英文界面
- 日文界面

### 重点关注
- 新增按钮是否有缺失翻译。
- 删除确认、导出/删除批量按钮、障碍导入提示是否正常翻译。
- 地图设置、模板 JSON、实验记录、任务队列区域是否有明显排版畸形。

## 第八组：封装模式终验

### 场景 A：打包产物启动
1. 运行 `dist/AGV_Dispatch_Package/start_agv.bat`。
2. 确认浏览器自动打开。
3. 确认页面与 API 都可访问。

### 场景 B：封装模式基础回归
至少手工确认：
- 页面可进入
- 地图正常显示
- 点位 / 模板 / 障碍预设可见
- 自动单段任务可跑通

## 终验结论写法建议

### 可直接标记为通过的条件
- 没有阻断型问题。
- 启动、调度、异常、数据、封装五大块都能完成核心演示。

### 建议标记为“通过但有观察项”的条件
- 只有轻微 UI 问题。
- 不影响演示流程、不影响答辩说明。

### 建议标记为不通过的条件
- 任一启动方式无法用。
- 自动 / 手动 / 多段任务主链断掉。
- SQLite / MySQL / 封装模式任一出现阻断。
