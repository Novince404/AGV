# 个人端答辩演示操作手册

## 1. 推荐演示顺序

1. 双击 `start_agv.bat` 启动系统。
2. 使用个人用户登录，推荐账号为 `personal_demo` / `personal123`。
3. 打开地图设置，导入 `demo/json/personal_defense_map_profile_10x8.json`。
4. 在地图方案列表中点击“预览”，确认是 10x8 货架通道图。
5. 点击“应用”，让当前地图切换到个人答辩演示地图。
6. 回到主界面，确认 3 台默认 AGV 可见，且常用点位没有被障碍物覆盖。
7. 打开 JSON 导入/导出，选择 `demo/json/personal_defense_busy_tasks.json` 导入。
8. 观察地图上多台 AGV 同时运行、剩余任务排队、路径绕开障碍物。

## 2. 备用障碍物恢复方式

如果不想使用地图 Profile，可以只导入障碍物：

1. 打开地图设置。
2. 导入 `demo/json/personal_defense_obstacle_layout_10x8.json`。
3. 点击“保存障碍”。
4. 再导入 `demo/json/personal_defense_busy_tasks.json`。

注意：任务导入会校验起终点是否被障碍物占用，所以必须先应用配套地图或保存配套障碍物。

## 3. 演示观察点

- 默认 3 台 AGV 会先接收前几个任务。
- 后续任务会留在队列中，等待 AGV 空闲后继续调度。
- 地图中的竖向障碍列模拟货架，路径会从预留通道绕行。
- 多阶段任务会显示阶段进度，适合说明“入库、暂存、装配、出库”的业务链路。
- 打开常用点位显示时，可以更清楚地解释入库口、出库口、存储位等业务语义。
- 关闭常用点位显示时，地图更适合观察 AGV 路线和拥堵状态。

## 4. 建议讲解词

“我这里准备了一组批量任务，不是逐个手动创建。导入后系统会自动进入任务队列，并把当前空闲车辆分配出去。地图上的障碍物代表货架和不可通行区域，所以路径规划必须绕行。随着前面任务完成，队列里的任务会继续被接走，这能体现系统的连续调度能力。”

## 5. 现场兜底

- 如果导入任务失败，先检查是否应用了 `个人端答辩演示地图 10x8`。
- 如果车辆不动，检查 AGV 是否处于空闲、维护、故障或不可调度状态。
- 如果演示太快结束，可以再次导入同一个任务 JSON，让队列重新变忙。
- 如果界面信息太密，先关闭常用点位显示，只保留路线和 AGV 状态。
