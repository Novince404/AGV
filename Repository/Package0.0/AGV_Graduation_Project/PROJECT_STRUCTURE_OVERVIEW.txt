二、后端结构与职责（FastAPI）
-----------------------------
1. backend/main.py
- 后端入口文件。
- 创建 FastAPI 应用。
- 配置 CORS（允许前端 localhost:5173 访问）。
- 注册路由：AGV、Task、Schedule、Status、Fault。

2. backend/app/api/
- agv_api.py
  1) 维护 AGV 列表（当前内存对象）。
  2) 提供 AGV 列表查询。
  3) 提供 AGV 急停/恢复接口。
  4) 急停时会联动任务状态（必要时打回 blocked）。

- task_api.py
  1) 维护任务列表（当前内存对象）。
  2) 支持单段/多段任务创建。
  3) 支持任务导入导出（JSON）。
  4) 支持删除待分配/不可达任务。
  5) 包含任务阶段字段同步、坐标合法性与障碍校验调用。

- schedule_api.py
  1) 调度核心入口。
  2) 支持自动调度（按优先级+距离选任务与AGV）。
  3) 支持手动绑定（指定任务或指定AGV）。
  4) 路径规划调用 simple/A*。
  5) 输出 path_to_start/path_to_end。
  6) 支持算法对比 compare_path。
  7) 支持 blocked 任务改用 A* 重试。

- status_api.py
  1) 提供 AGV/Task 状态色板。
  2) 提供地图障碍查询、更新、预设切换、重置。
  3) 更新障碍时会过滤 AGV 当前占位，避免障碍覆盖车辆位置。

- fault_api.py
  1) 提供故障上报接口。
  2) 提供故障列表查询与故障解除接口。
  3) 与 AGV 状态、任务 blocked 状态联动。

3. backend/app/models/
- agv.py
  AGV 数据模型（id, x, y, status, task_id, active_fault_event_id）。

- task.py
  Task 与 TaskStage 数据模型。
  支持多段任务字段：current_stage_index、total_stages、stages、dispatch_algorithm、dispatch_mode 等。

- fault_event.py
  FaultEvent 数据模型。
  含 event_type、severity、status、reported_at、resolved_at 等。

4. backend/app/utils/
- path_planner.py
  路径规划算法实现：
  1) simple（轴向曼哈顿两种顺序尝试）
  2) A*（四邻域）
  并统一通过 plan_path 对外提供调用。

- agv_movement.py
  AGV 线程运动模拟：
  1) 按路径逐步移动。
  2) 支持多段任务阶段推进。
  3) 含碰撞占位保护（目标格被占时等待）。
  4) 处理 fault/emergency_stop 中断。

- warehouse_map.py
  地图网格尺寸、默认障碍、障碍预设管理。
  支持当前障碍集合读写与预设输出。

- task_chain.py
  多段任务工具层：阶段构建、当前阶段同步、路径写入、阶段推进、任务 blocked/pending 状态切换。

- fault_state.py
  故障事件内存存储与生命周期管理（创建、查询、按状态筛选、解除）。

- status_map.py
  AGV/TASK 状态颜色定义。

- api_error.py
  统一错误输出封装（error_code + 细节字段）。

三、已实现后端接口（当前）
-------------------------
1) AGV
- GET /agv/list
  功能：获取 AGV 列表。
- POST /agv/{agv_id}/emergency-stop
  功能：急停 AGV，必要时中断其任务。
- POST /agv/{agv_id}/resume
  功能：恢复急停 AGV。

2) Task
- GET /task/list
  功能：获取任务列表。
- POST /task/create
  功能：创建任务（单段或多段）。
- POST /task/finish/{task_id}
  功能：手动结束运行任务。
- POST /task/import_json
  功能：批量导入任务。
- GET /task/export_json
  功能：导出任务。
- DELETE /task/{task_id}
  功能：删除待分配/不可达任务。

3) Schedule
- POST /schedule/
  功能：默认 simple 自动调度一次。
- POST /schedule/with_path
  功能：按参数调度并返回路径。
- POST /schedule/compare_path
  功能：simple 与 A* 对比。
- POST /schedule/retry_blocked/{task_id}
  功能：blocked 任务改用 A* 重试（有空闲车则立即调度，否则入待分配）。

4) Status/Map
- GET /status/agv
- GET /status/task
- GET /status/map
- GET /status/map/presets
- PUT /status/map
- POST /status/map/preset/{preset_key}
- POST /status/map/reset

5) Fault
- POST /fault/report
- GET /fault/list
- POST /fault/{event_id}/resolve

四、前端结构与职责（Vue3 + Vite）
-------------------------------
1. src/main.js
- 前端入口，挂载 App.vue。

2. src/App.vue（主页面编排层）
- 负责页面整体布局与状态编排。
- 对接后端接口（任务、调度、地图、故障、AGV）。
- 聚合 composables 与 utils，完成：
  1) 地图渲染与交互
  2) AGV 显示与选车
  3) 自动/手动调度触发
  4) 任务创建（单段、多段、模板、JSON）

  7) 多语言显示

3. src/assets/agv-map.css

  单段/多段任务构建器状态管理与地图选点流程控制。

7. src/utils/

- taskChain.js
  多段任务前端数据结构工具（阶段构造、默认链、起终点解析等）。

  下载工具（JSON/CSV）。

五、当前核心运行流程（简述）
--------------------------
1) 前端创建任务（单段/多段） -> /task/create
2) 调度触发 -> /schedule/with_path
3) 后端按算法生成 path_to_start + path_to_end
4) 后端线程 move_agv 按路径推进 AGV 坐标
5) 前端轮询 /agv/list + /task/list 刷新状态与路径展示
6) 遇到不可达则任务进入 blocked，可改用 A* 重试
7) 故障/急停事件可中断任务并进入 fault 流程

六、运行入口与启动方式
----------------------
1) 推荐启动脚本：run_dev.bat
- 自动启动 backend（uvicorn）
- 自动启动 frontend（vite）
- 自动尝试打开浏览器到 http://localhost:5173/

2) 后端单独启动
- 进入 backend，激活 venv
- 执行：python -m uvicorn main:app --reload

3) 前端单独启动
- 进入 frontend/agv-frontend
- 执行：npm run dev
