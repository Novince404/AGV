# 第四阶段实施计划 v4：企业版二代路网、角色工作台与双反馈体系

## 一、摘要
- 本阶段只针对**企业版升级**，不重构个人用户现有显示、个人主界面和默认交互。
- 个人用户在本阶段只新增一项能力：**向平台管理员发送 bug 反馈**。
- 企业版将从“显示全部功能框 + 无权限时禁用/只读提示”升级为：**企业角色只看到自己可操作的主面板和企业设置分栏**。
- 第四阶段除企业版路网/连续运动主线外，同时新增两套独立反馈能力：
  - `企业内部请求/报错`
  - `平台 bug 反馈`
- 平台管理员继续保持**治理优先首页**，新增 bug 反馈处理入口，但不恢复默认完整地图与任务栏首页。

## 二、关键改动

### 4A：企业角色工作台重构与反馈基础
- 企业前端按角色做**整块隐藏**：
  - 主控制台只保留该角色可操作的面板。
  - 企业设置只保留该角色可进入的分栏。
  - 不再以大块禁用/只读卡片作为主呈现。
- 建立企业版角色工作台映射：
  - `enterprise_admin`
  - `enterprise_operator`
  - `enterprise_logistics`
- 每个角色明确对应：
  - 可见主面板
  - 可见企业设置分栏
  - 可发起的反馈类型
- 新增两套反馈模块：
  - `企业内部请求/报错`
    - 仅企业角色可用
    - 发送时可选择企业内接收对象
    - 接收对象范围限定为企业内部用户，不含平台管理员
  - `平台 bug 反馈`
    - 企业角色和个人用户都可用
    - 统一发送给平台管理员
- 平台管理员首页新增治理关联入口：
  - 企业审批
  - 账号治理
  - 平台 bug 反馈
  - 必要审计摘要
- 第一刀先做最小可用单据模型：
  - 标题
  - 分类
  - 内容
  - 提交人
  - 接收人/目标端
  - 状态
  - 创建时间
  - 最近更新时间

### 4B：企业版路网拓扑与地图/路线底层模型升级
- 在现有 `valid_cells / blocked_cells` 基础上新增企业版**拓扑层**，不推翻当前异形网格底座。
- 地图拓扑至少包含：
  - `nodes`
  - `edges`
  - `stations`
  - `parking_nodes`
  - `charge_nodes`
  - `topology_version`
- `edges` 第一版至少表达：
  - 起点 / 终点
  - 单向 / 双向
  - 车道类型
  - 长度或权重
  - 默认速度倍率
- 旧地图继续兼容：
  - 无拓扑时仍按旧网格运行
  - 企业版带拓扑时启用新模式
- 企业设置新增企业版路网入口，但只在企业可用角色中显示；个人用户不显示这套能力。

### 4C：连续移动与时间步推进
- AGV 从“格子跳点”升级为“沿边连续推进”。
- AGV 状态新增：
  - `current_node`
  - `current_edge`
  - `edge_progress`
  - `motion_state`
  - `current_speed`
  - `target_speed`
  - `heading`
- 企业版地图支持连续位置插值显示 AGV。
- 保留旧网格路径模式作为兼容回退，便于平滑迁移和排障。
- 个人用户显示逻辑保持当前模式，不跟随第四阶段进行连续运动式重构。

### 4D：冲突预占、双车道与优先级避让
- 在企业版拓扑边/节点上引入预占机制。
- 第一版支持：
  - 节点占用冲突
  - 对向边冲突
  - 同边冲突
  - 目标点占用
- 双车道放在这一包实现，不在 4B 提前落地：
  - 路网可表达双车道
  - 调度可识别可用车道
  - 避让可选择等待或切换边
- 第一版优先级固定：
  - 有任务车 > 空闲车
  - 装卸 / 停靠中 > 普通过路
  - 同优先级按任务创建时间与 AGV 编号稳定排序

### 4E：回仓、回充与空闲策略
- 新增企业版节点类型：
  - `parking`
  - `charge`
- AGV 状态机新增：
  - `idle_returning`
  - `charging`
  - `waiting_for_charge`
- 第一版规则固定：
  - 空闲超时回仓
  - 电量低于阈值回充
  - 新任务优先于空闲回仓

### 4F：企业独立 EXE 客户端（后置）
- 不放进第四阶段第一刀。
- 只在 4A ~ 4E 稳定后，再做企业独立客户端壳层、默认配置和资源裁剪。
- 当前导师演示包继续保留，但不等于企业独立 EXE 的正式交付。

## 三、公共接口与类型变化
- 地图模型扩展但兼容旧字段：
  - 保留网格字段
  - 新增 `topology` 或等价对象
  - 新增模式区分，如 `grid` / `topology`
- 企业反馈新增两套实体：
  - `enterprise_request`
  - `platform_bug_feedback`
- 反馈实体至少共享这些核心字段：
  - `id`
  - `type/category`
  - `title`
  - `content`
  - `submitter`
  - `target`
  - `status`
  - `created_at`
  - `updated_at`
- AGV 状态接口新增连续运动字段：
  - `position`
  - `heading`
  - `current_edge`
  - `edge_progress`
  - `motion_state`
- 企业角色可见性需要新增前端映射或后端返回能力摘要，用于决定哪些整块面板直接隐藏。

## 四、实施顺序
1. 先落正式 `md / docx`
2. 实现 `4A`
   - 企业版角色整块隐藏
   - 企业内部请求 / 报错
   - 平台 bug 反馈
   - 平台管理员 bug 反馈入口
3. 实现 `4B`
   - 企业版路网拓扑层
4. 实现 `4C`
   - 连续移动与时间步推进
5. 实现 `4D`
   - 双车道与预占避让
6. 实现 `4E`
   - 回仓 / 回充
7. 最后再评估并启动 `4F`

## 五、测试计划
- 企业三个角色登录后，只看到自己可操作的主面板和企业设置分栏；无权限整块不出现。
- 个人用户界面无第四阶段式布局回归，但可提交平台 bug 反馈。
- 企业用户可创建企业内部请求/报错，并能选择企业内部接收对象。
- 企业用户和个人用户都可提交平台 bug 反馈，平台管理员能在治理优先首页处理中看到。
- 平台管理员默认首页仍不显示完整地图与任务栏，只有显式进入测试视图时才出现。
- 旧矩形地图和现有异形地图在无拓扑时继续可用。
- 企业版带拓扑地图可保存、加载、运行。
- 连续移动模式下 AGV 不再表现为格子瞬移。
- 双车道场景下可触发基础等待或让行。
- 回仓 / 回充规则在企业版可自动触发。

## 六、默认假设
- 第四阶段继续以当前 `valid_cells` 异形网格为地图底座，不做真正自由多边形地图。
- 企业内部请求/报错与平台 bug 反馈是**两套独立模块**，不合并成统一工单系统。
- 企业内部请求第一版允许选择企业内部接收对象，但不做复杂会签、抄送和多级流转。
- 平台 bug 反馈第一版统一由平台管理员处理，不再细分到更复杂的组织流转。
- 第四阶段所有新 UI 和新模式默认只落在企业角色分支内；个人用户只增加平台 bug 反馈入口，不改主界面结构。
- 本计划对应的正式文档文件为：
  - `PROJECT_IMPLEMENTATION_PLAN_v4.md`
  - `PROJECT_IMPLEMENTATION_PLAN_v4.docx`
