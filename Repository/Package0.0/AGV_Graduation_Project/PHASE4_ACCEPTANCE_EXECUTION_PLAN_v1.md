## 5. 本轮执行原则

- 优先按 `PHASE4_ENTERPRISE_ACCEPTANCE_CHECKLIST.md` 推进，不重做已稳定模块
- 先补基线健康检查，再进高优先级验收链
- 合站统一语义放在验收主链之后收口，不抢占高风险逻辑修复优先级
- 默认不新增后端 API、数据库表或真正“站级调度实体”
- 保持“展示合站、调度按节点”的低风险策略

## 6. 执行顺序

### 第一阶段：落根目录计划文件
- 新增本文件，固化当前执行计划

### 第二阶段：补齐基线健康检查
- `backend\\venv\\Scripts\\python.exe -m compileall backend\\app`
- `backend\\venv\\Scripts\\python.exe backend\\scripts\\sqlite_smoke_check.py`
- `frontend\\agv-frontend\\npm run lint`
- `frontend\\agv-frontend\\npm run build`

通过标准：

- 上述 4 项全部通过
- 若任一失败，先修复基线，再继续验收推进

### 第三阶段：按验收清单推进高优先级链路

#### A. 账号与数据隔离
- 个人地图与企业地图隔离
- 个人 AGV 与企业 AGV 隔离
- 企业管理员/操作工/后勤岗共享同一企业数据

#### B. 企业独立客户端 4F
- 三角色登录链
- `fail to fetch`
- 地图、AGV、任务、企业设置等关键入口可用

#### C. 拓扑与运行链 4B/4C
- 拓扑保存链
- 拓扑运行链
- 路径显示时机与任务完成后的清理时机

#### D. 冲突与避让 4D
- 同路不超车
- 双向相遇避让
- 死锁打破

#### E. 回仓回充 4E
- 充电站多车容量
- 等待充电状态
- 站内统计与站内选车逻辑

### 第四阶段：按验收清单推进低风险 UI/交互链
- 小铃铛未读/待处理状态
- 说明中心与设置弹窗层级
- 站点、停车点、充电点的 `n/m` 与悬停可读性

### 第五阶段：聚合站点统一语义收口
- 运行态主地图上的聚合停车站/充电站统一为一个站级按钮
- 主地图只显示站点身份与一个主 `n/m`
- 内部泊位信息只在详情层展示
- 拓扑编辑器继续保留点位级可编辑语义
