# 企业端动态避让增强计划

## 目标

企业端动态避让定位为调度层能力，重点处理拓扑边、交汇节点、站点容量和任务优先级冲突。它不替代真实机器人传感器级避障，也不引入激光雷达、视觉、DWA、ORCA 或 MPC 等连续空间局部避障算法。

本轮增强按“验收基线 -> 可视化 -> 运行时预约”的顺序推进，避免破坏已经稳定的企业拓扑、停车站、充电站、回仓和回充能力。

## 当前基线

- 企业端已有拓扑边运行时占用保护。
- 已有同廊道跟车距离判断。
- 已有节点容量与站点占用判断。
- 已有等待、让行、重规划和死锁超时重试。
- 个人端已切换为纯网格动态避让，不应把个人端让行格规则套到企业端拓扑上。

## 本轮实施

- 新增企业端演示资产，覆盖单通道对向会车、十字交汇三车抢占、站点出入站冲突。
- 增强企业端运行时资源预约：车辆进入拓扑边前，同时预约拓扑边和容量为 1 的目标节点。
- 预约表只保存在内存中，不新增数据库表，不改变已有公开 API。
- 前端继续复用 `motion_state`、`current_edge` 和 `dispatch_reason`，把企业端冲突原因显示成更可读的“路段占用 / 交汇节点预约 / 跟车等待 / 重规划”。

## 验收场景

1. 单通道对向会车：两台车从同一拓扑边两端相向运行，低优先级车辆等待或重规划，不能同时占用主边。
2. 十字交汇三车：三台车从不同方向接近中心节点，中心节点一次只能被一台车预约，后续车辆等待或重规划。
3. 站点出入站冲突：停车点和充电点容量保持有效，出入站车辆不能破坏站内容量和选车逻辑。

## 后续方向

- 若三车或更多车辆仍偶发局部僵持，再扩展为多步时间窗 reservation table。
- 若接入真实设备，再在调度层之下叠加传感器级局部避障。
