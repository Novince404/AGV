# 个人端导师演示与答辩方案 2026-04-19

## 1. 演示目标

这次演示建议把重点放在“个人用户也能完成一套小型 AGV 调度闭环”上，而不是把所有后台治理和企业端能力都铺开讲。导师需要快速看到三个层次：

- 系统能建模：地图、障碍物、常用点位、任务起终点。
- 系统能调度：一次导入多任务后，多个 AGV 同时执行，剩余任务排队接续。
- 系统能解释：任务状态、路径、算法、日志、导入导出和异常兜底都有可观察入口。

推荐演示时长控制在 10 到 15 分钟。先展示个人端调度主流程，再用 2 到 3 分钟解释系统架构与关键设计，最后预留提问。

## 2. 演示前准备

1. 启动项目：双击 `start_agv.bat`，等待浏览器打开并完成登录页加载。
2. 建议使用个人演示账号：`personal_demo` / `personal123`。如果本机数据库已换过账号，可以使用任意个人用户登录。
3. 登录后先确认地图是 `10 x 8`，默认 3 台 AGV 都处于空闲或可调度状态。
4. 打开地图设置，导入 `demo/json/personal_defense_map_profile_10x8.json`，然后应用该地图方案。
5. 如果只想临时恢复障碍物，不保存地图 Profile，也可以导入 `demo/json/personal_defense_obstacle_layout_10x8.json`，导入后点击“保存障碍”。
6. 进入 JSON 工具，准备导入 `demo/json/personal_defense_busy_tasks.json`。

## 3. 现场演示脚本

### 开场 1 分钟

可以这样讲：

“这个系统面向 AGV 仓储调度场景，个人端主要用于小规模仿真、算法演示和任务调度验证。今天我先用一个 10x8 的仓储通道地图，演示障碍物建模、多任务导入、自动派车、路径规划和任务排队。”

### 地图与障碍物 2 分钟

展示配套地图时，重点说明：

- 黑色或阻塞格模拟货架、墙体或不可通行区域。
- 障碍物不是装饰，任务导入和路径规划都会校验它。
- 当前地图保留了几个业务常用点位：入库口、出库口、存储位、装配位、质检台、维护位。
- 可以打开或关闭“常用点位显示”，让界面在讲业务时更清晰，在看调度时更简洁。

### 多任务导入 3 分钟

导入 `personal_defense_busy_tasks.json` 后，建议这样讲：

“这里不是手动一个个点任务，而是一次导入一组业务任务。系统会自动把任务写入队列，并尝试分配给当前空闲 AGV。默认 3 台车会先接走 3 个任务，剩下任务保留在待执行队列中，等车辆完成后继续接续。”

观察重点：

- 多台车同时运行，路线会显示在地图上。
- 部分任务处于待执行或等待空闲 AGV，体现队列调度。
- 多阶段任务会按“入库、跨区转运、出库”顺序执行。
- A* 路径会绕开货架障碍，不是直线穿过。

### 调度解释 2 分钟

可以从界面状态说到后端逻辑：

- 任务状态从 pending 到 assigned/running，再到 finished。
- AGV 有电量、位置、任务绑定、运行状态等字段。
- 地图障碍物影响任务合法性和路径搜索。
- 自动派车会考虑车辆是否空闲、任务优先级、距离与算法配置。

### 功能补充 2 分钟

根据导师兴趣选择展示：

- 左键地图点位设置起终点，右键查看点位或站点详情。
- H 说明中心可以按个人端/企业端分组折叠查看。
- JSON 导入导出支持复现实验场景。
- 异常任务可以删除、恢复或重新导入演示。

## 4. 答辩讲解口径

### 为什么要做 JSON 多任务导入

手动创建任务适合单次演示，但答辩时更重要的是证明系统具备批量调度能力。JSON 导入可以把同一批任务、同一张地图、同一组障碍物复现出来，方便对比算法效果，也方便导师现场重复验证。

### 为什么选择 10x8 地图

10x8 足够小，导师一眼能看懂；同时又能放下货架通道、多个业务点位和 3 台以上 AGV 的运动路径。它适合演示系统能力，而不是用过大的地图让现场观察变困难。

### 常用点位和停车/充电站的区别

常用点位是业务地标，例如入库口、出库口、存储位、装配位。它主要服务任务创建和讲解。

停车点、充电点、停车站和充电站属于 AGV 运行资源，它们参与回仓、回充、容量和可调度状态。演示中要避免把“充电区”当成业务常用点位继续使用，防止和真实充电站语义重叠。

### 算法层面可以怎么讲

当前演示重点是基于网格地图的路径规划与任务调度。障碍物会进入路径规划约束，AGV 按阶段推进，任务完成后释放车辆。企业端进一步扩展了路网拓扑、站点容量、回仓回充和角色隔离。

## 5. 演示文件说明

- `demo/json/personal_defense_map_profile_10x8.json`：推荐优先使用，导入后形成可复用地图方案。
- `demo/json/personal_defense_obstacle_layout_10x8.json`：只恢复障碍物布局时使用，导入后记得保存障碍。
- `demo/json/personal_defense_busy_tasks.json`：个人端答辩用批量任务，一次导入后让多台车同时忙起来；坐标已避开默认障碍和配套演示障碍，适合作为现场兜底版本。
- `demo/docs/PERSONAL_DEFENSE_DEMO_RUNBOOK.md`：现场逐步操作手册。

## 6. 现场兜底方案

- 如果地图 Profile 导入后没有立即切换，先在地图方案列表中点击“应用”。
- 如果任务导入提示路径或阶段被障碍阻塞，说明当前地图不是配套地图，重新应用 `personal_defense_map_profile_10x8.json`。
- 如果只有一两台车动起来，先确认 AGV 是否处于空闲状态；演示效果仍然成立，因为剩余任务会进入队列等待空闲车。
- 如果现场误操作导致任务太多，可以清理已完成任务或刷新后重新导入演示 JSON。
- 如果浏览器出现卡顿，优先关闭不必要的弹窗和说明中心，再保留主地图、任务列表和 AGV 列表三个区域。
