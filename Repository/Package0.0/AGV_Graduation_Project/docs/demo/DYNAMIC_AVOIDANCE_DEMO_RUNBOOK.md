# 动态避让演示操作说明

## 目标

这组素材用于稳定展示“调度层动态避让”，重点不是让 AGV 随机绕障，而是让多台车在同一窄通道和交汇口里产生可观察的占用冲突，再看系统如何等待、让行、重试和恢复。

配套文件：

- `demo/json/dynamic_avoidance_map_profile_12x8.json`
- `demo/json/dynamic_avoidance_conflict_tasks.json`

## 推荐演示顺序

1. 确认当前没有正在运行的任务。若任务队列里有旧任务，先等待完成或清理已完成任务。
2. 在 JSON / 地图工具中导入 `dynamic_avoidance_map_profile_12x8.json`。
3. 在地图方案列表中应用“动态避让窄通道演示地图 12x8”。
4. 确认至少有 3 台空闲 AGV。默认三台 AGV 的初始位置已经适合触发冲突：`(2,3)`、`(5,7)`、`(1,4)`。
5. 导入 `dynamic_avoidance_conflict_tasks.json`。
6. 观察中部窄通道和十字交汇点：车辆应在同一走廊、对向边或中心节点附近出现等待、让行、恢复继续运行。

## 观察重点

- 同一物理走廊里，后车不应直接穿过前车。
- 对向车辆进入中部窄通道时，应出现一方等待或重规划，而不是两车互相穿透。
- 南北方向任务经过中心交汇点时，应受到节点占用约束。
- 低优先级补充任务在没有空闲 AGV 或前方占用时可以排队，前序任务释放后继续派发。
- 任务完成后，AGV 应回到空闲状态，不应残留 `current_edge`、旧路径或任务绑定。

## 如果现场没有触发明显避让

- 先确认任务算法是 `astar`，配套任务 JSON 已经写入该字段。
- 先确认地图确实应用了 `12 x 8` 动态避让地图，而不是只导入但未应用。
- 如果当前 AGV 位置已经被前一次演示改变，可以先创建或移动几台空闲车到走廊两端附近，再重新导入任务。
- 如果系统正在自动回仓或回充，先让这些自治动作结束，避免演示任务被回仓 / 回充行为稀释。

## 能力边界

这套演示展示的是调度层动态避让：系统基于离散网格、拓扑节点、拓扑边、容量和任务优先级处理冲突。它不等同于真实机器人传感器级避障，也不包含连续空间速度规划。后续接入真实设备时，可以在这个调度层基础上继续叠加激光雷达、视觉、DWA、ORCA、MPC 等局部避障能力。
