# 发布说明与版本策略

最后更新：2026-03-18

## 目标
- 让版本号、Release、提交节奏更稳定
- 方便毕业设计阶段展示、周报记录、导师沟通与后续封装交付

## 当前版本策略

### 正式版
- 适合：
  - 一个大阶段已经稳定
  - 核心功能、文档、演示流程基本完整

示例：
- `v1.5.0`
- `v1.6.0`

### Beta 版
- 适合：
  - 架构升级较大
  - 已能演示，但仍在联调
  - 可能还有交互或边角问题待收口

示例：
- `v2.0.0-beta.1`

## 推荐的语义

### 补丁版本
- 只修 bug
- 例如：
  - `v1.6.1`

### 次版本
- 增加一批稳定功能
- 例如：
  - `v1.7.0`

### 主版本
- 架构或产品阶段有明显跃迁
- 例如：
  - 接入数据库
  - 完成封装
  - 引入登录和角色体系

## 当前推荐节奏

### 现在
- 保持：
  - `v2.0.0-beta.x`
- 用于数据库接入、封装验证、最低可交付收口阶段

### 等满足以下条件后，再转正
- SQLite 演示稳定
- 封装包稳定
- 演示脚本与测试清单完整
- 核心链路回归通过

届时可升级为：
- `v2.0.0`

## GitHub Release 处理建议

### 什么时候发 draft release
- 当一个阶段已经成型
- 即使还没完全转正，也可以先发 draft

### Release 文案建议包含
- Highlights
- Added
- Improved
- Fixed
- Validation
- Notes

## 提交信息建议

### 功能
- `feat: ...`

### 修复
- `fix: ...`

### 重构
- `refactor: ...`

### 文档
- `docs: ...`

### 阶段版本
- `release:vX.Y.Z`

## 当前最低可交付版建议
- 开发阶段标签：`v2.0.0-beta.x`
- 最低可交付版完成后：
  - 升为 `v2.0.0`

## 与文档联动
- 每次版本更新建议同步检查：
  - `CHANGELOG.md`
  - `RELEASE_DRAFT_*.md`
  - `RELEASE_UPLOAD_*.md`
  - `PROJECT_IMPLEMENTATION_PLAN_v2.0.md`
