# 平台账号治理增强计划 v1

## 目的
- 本计划是对当前 `v3` 主线的补充，不替代 `PROJECT_IMPLEMENTATION_PLAN_v3.md`。
- 本计划聚焦“个人注册 + 平台管理员账号治理”能力。
- 本计划完成后，继续回到既定大主线推进。

## 目标
- 新增个人用户注册入口，个人注册后可直接登录，不进入审批流。
- 平台管理员可查看新注册的个人用户与现有企业账号。
- 平台管理员可对个人用户与企业账号执行封禁、解封、停用/删除等治理动作。
- 封禁动作支持备注原因、封禁时长、是否永久封禁。
- 用户登录时可看到自己的受限状态、原因、时长与备注。
- 平台管理员可导出账号治理关键数据。

## 推荐范围
### 本期纳入
- 个人用户注册。
- 平台管理员账号管理列表页。
- 账号级封禁与解封。
- 账号级软删除/停用。
- 登录态联动提示。
- 审计与导出。

### 本期不做
- 组织级整体封禁。
- 企业组织内全员联动停用。
- 真正物理删除数据库账号记录。
- 更复杂的风控策略、验证码、人机校验。
- 邮件/短信通知。

## 关键原则
- 个人注册直接成功，不进入审批。
- 企业注册仍保留原有审批流，不被本计划替代。
- 删除优先做“软删除/停用”，不做物理删除。
- 企业账号治理第一版先做“账号级”，不直接升级成“组织级”。
- 所有治理动作必须进入操作审计。

  - `suspended`
  - `deactivated`
- `suspension_reason`
- `suspension_note`

- `suspended_until`
- `suspended_by`

### 审计实体补充方向
- `resource_type = user_account`
- `action_type`
  - `user.register.personal`
  - `user.suspend`
  - `user.unsuspend`
  - `user.deactivate`

## 接口建议
### 个人注册
- `POST /auth/register-personal`

### 平台管理员用户治理
- `GET /auth/users`
- `GET /auth/users/{id}`
- `POST /auth/users/{id}/suspend`
- `POST /auth/users/{id}/unsuspend`
- `POST /auth/users/{id}/deactivate`
- `GET /auth/users/export`

### 登录态与当前会话
- `GET /auth/me`
  - 增补返回：
    - `account_status`
    - `suspension_reason`
    - `suspension_note`
    - `suspended_until`
    - `governance_updated_at`

## 实施拆分
### 板块 A：个人注册
- 登录弹窗新增个人注册页。
- 后端新增 `register-personal`。
- 成功后直接获得个人身份。

### 板块 B：平台管理员账号治理工作台
- 新增平台管理员入口。
- 用户列表、筛选、详情、导出。

### 板块 C：封禁/解封/停用
- 新增治理动作接口。
- 动作写入审计。
- 列表和详情状态联动。

### 板块 D：登录联动与状态反馈
- 登录拦截被封禁/停用账号。
- 登录页展示原因、备注、封禁到期时间。
- 已登录状态刷新时同步治理状态。

## 测试清单
### 个人注册
- 新个人账号注册成功。
- 注册后可直接登录。
- 不进入企业审批流。

### 平台管理员账号治理
- 可查看个人用户与企业账号。
- 可按角色/状态筛选。
- 可查看账号详情。
- 可导出关键数据。

### 封禁与解封
- 封禁后账号无法正常登录。
- 登录页可看到封禁原因与时长。
- 到期前保持封禁。
- 解封后恢复登录。

### 停用/删除
- 停用后账号不可继续使用。
- 审计记录仍保留。
- 列表中状态正确变化。

### 审计
- 注册、封禁、解封、停用、导出均进入审计。

## 风险与边界提醒
- 若后续要做“企业组织级封禁”，会牵涉企业三岗位联动与已登录会话统一失效，不建议在本计划第一版混做。
- 若后续要做“物理删除”，需要额外处理审计、素材、模板、实验记录、任务记录的关联保留策略。
- 本计划建议优先做 MySQL / SQLite 双兼容实现，保持现有环境一致性。

## 结论
- 本计划适合作为当前 `v3` 主线之外的一次平台账号治理增强。
- 第一版建议先收个人注册、平台账号治理工作台、账号级封禁/停用、登录状态联动与导出。
- 完成后，再回到既定大主线继续推进。
