# 第四阶段企业版二代回归检查表

这份清单用于第四阶段总收口：先确认 4A-4F 的参数、交互、运行链没有被后续修改打断，再进行导师演示或企业独立客户端演示。

## 0. 启动前健康检查

在仓库根目录运行：

```bat
backend\venv\Scripts\python.exe -m compileall backend\app
backend\venv\Scripts\python.exe backend\scripts\sqlite_smoke_check.py
backend\venv\Scripts\python.exe backend\scripts\runtime_conflict_smoke.py
backend\venv\Scripts\python.exe backend\scripts\task_json_import_smoke.py
backend\venv\Scripts\python.exe backend\scripts\enterprise_client_login_smoke.py
backend\venv\Scripts\python.exe backend\scripts\feedback_notification_smoke.py
cd frontend\agv-frontend
npm run lint
npm run build
```

独立客户端包验收时，双击：

```bat
dist\AGV_Enterprise_Client_v1\start_enterprise_client.bat
```

通过标准：

- 页面可以打开企业独立客户端登录页。
- 登录后不出现 `fail to fetch`。
- 后端窗口持续运行，没有连续刷 500 报错。

## 1. 演示账号

预置演示账号：

- 个人用户：`personal_demo` / `personal123`
- 企业管理员：`enterprise_demo` / `enterprise123`
- 企业操作工：`enterprise_operator_demo` / `operator123`
- 企业后勤岗：`enterprise_logistics_demo` / `logistics123`
- 平台管理员：`admin_demo` / `admin123`

企业独立客户端只验企业管理员、企业操作工、企业后勤岗。

## 2. 账号与数据隔离

网页端回归：

1. 登录个人用户，修改地图，例如切换地图方案或保存一处障碍。
2. 登出并登录企业管理员。
3. 确认企业地图没有被个人用户的修改污染。
4. 在企业管理员下新增或调来 AGV。
5. 登出并登录个人用户。
6. 确认个人 AGV 列表没有混入企业 AGV。

通过标准：

- 个人用户与企业组织的 AGV 数据分开。
- 个人用户与企业组织的地图数据分开。
- 企业管理员、企业操作工、企业后勤岗共享同一企业组织数据。

## 3. 企业独立客户端登录链

分别登录：

- 企业管理员
- 企业操作工
- 企业后勤岗

每个账号都检查：

- 顶部身份和权限说明正确。
- 地图正常加载。
- AGV 列表正常加载。
- 任务列表正常加载。
- 企业设置或对应岗位入口可以打开。

如果任意账号显示 `fail to fetch`，先停止演示，检查企业客户端后端窗口、端口、数据库连接。

## 4. 派发任务与路线显示

建议先用企业管理员或企业操作工验收：

1. 新建或选择一台空闲 AGV。
2. 在自动模式下点选终点并派发任务。
3. 观察 AGV 开始动的第一时间。
4. 等待任务完成。

通过标准：

- 派发后马上能看到起点、终点、规划路径，不应该等 AGV 走出两格才显示。
- AGV 开始移动前后，路径叠加层不应该闪一下就消失。
- 到达终点后，任务路径、起点、终点应在任务收尾时及时清理。
- 任务卡片状态和地图上的小车状态一致。

## 5. JSON 任务导入

在任务区域测试：

- 导入 `demo/json/task_auto_single_demo.json`
- 导入 `demo/json/task_manual_single_demo.json`
- 导入 `demo/json/task_multi_stage_demo.json`

通过标准：

- 自动任务导入后可以进入自动分配，不只是停在任务卡片里。
- 手动任务显示清楚的待指定车辆说明。
  当前 `task_manual_single_demo.json` 默认不预绑定 AGV，用于验证这一条。
- 多阶段任务导入不报失败。

## 6. 拓扑主干道

建议准备一条主干道，并把速度倍率临时调高到 4-5，便于肉眼观察。

检查：

- 起点和终点正好落在拓扑边端点。
- 起点或终点落在拓扑边中间。
- 起点和终点都不在拓扑边，但中途接入主干道再离开。

通过标准：

- 只要 AGV 正在拓扑边上，就按拓扑速度运行。
- 离开拓扑边后恢复基础速率。
- 不应强制跑完整条拓扑边之后再回头下主干道。
- 不应为了进最近拓扑节点而走明显更远的路线；如果主干道不省时间，可以直接走普通路。

## 7. 站点、停车点、充电点

检查：

- 站点、停车点、充电点可以显示容量。
- 多台 AGV 可以收纳到容量允许的站内。
- 入站后的单台小车图标不会遮挡站内 `n/m` 显示。
- 悬停能看到站内 AGV 编号。
- AGV 电量会显示。
- AGV 行驶、等待、停车、充电时，电量趋势符合当前配置。

## 8. 通知与协同

检查：

- 企业内部请求被创建后，收件岗位能看到提示。
- 平台 Bug 反馈被创建后，平台管理员能在治理侧或反馈入口看到。
- 地图右下角的小铃铛不遮挡页面返回顶部按钮。
- 小铃铛有未读/待处理变化时，视觉状态应有变化。

当前已补的自动化回归：

- `backend\scripts\feedback_notification_smoke.py`
- 已覆盖“企业内部请求创建 -> 目标岗位可见 -> 状态更新后企业管理员侧同步刷新”
- 已覆盖“个人/企业用户提交平台 Bug -> 平台管理员可见并处理 -> 提交者侧状态回刷”

## 9. 说明中心与设置弹窗

检查：

- 首次进入企业版会出现说明中心。
- 按 `H` 可以重新打开说明中心。
- 企业设置、页面设置打开后，说明中心仍在更上层。
- 说明中心里能读到企业拓扑主干道、基础段、等待/让行、回仓回充等说明。

## 10. 不能放过的失败现象

如果出现下面任一项，先记录复现步骤再继续改：

- 登录企业独立客户端后 `fail to fetch`。
- 切换账号后地图或 AGV 串到另一个账号。
- 派发任务后，任务有起终点但 AGV 不动。
- AGV 已动很久，路径才出现。
- 任务完成后，起点、终点、路径长期残留。
- 起终点不在拓扑节点上时，AGV 硬跑到拓扑终点再折返。
- 充电站容量大于 1 时，第二台车仍在外面一直紫色等待。
- 同一条物理走廊里，后车直接穿过前车。

## 11. 推荐演示顺序

1. 企业管理员登录，展示企业工作台和说明中心。
2. 打开企业设置，展示地图方案、拓扑编辑、节点/连边容量和倍率。
3. 派发一个普通自动任务，确认路线立即出现。
4. 派发一个会接入主干道的任务，解释“普通段 -> 主干道 -> 普通段”。
5. 展示停车点/充电点多车容量、电量、回仓/回充。
6. 切换企业操作工，展示同一企业数据和调度视角。
7. 切换企业后勤岗，展示地图/点位/模板维护视角。
8. 回到网页端个人用户，确认个人地图和企业地图互不污染。
